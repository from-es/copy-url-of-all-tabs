# ビルドとインストール

**最終更新日:** 2026年3月25日

## ビルド

[Node.js](https://nodejs.org/) がインストールされていることを前提としています。

1. このリポジトリをローカルにクローンするか、ファイルをダウンロードします。
2. 任意のローカルディレクトリにファイルを配置します。
3. `package.json` があるディレクトリに移動します。
4. 依存関係をインストールするために `npm ci` を実行します。

> [!IMPORTANT]
> **なぜ `npm install` ではなく `npm ci` なのか？**
> `npm ci` は `package-lock.json` を厳密に再現し、クリーンな状態でインストールを行います。これにより、環境による依存バージョンの不一致を防ぎ、ビルドの再現性と安定性を確保できます。
>
> **使い分けガイドライン:**
> - **`npm ci`**: 初回セットアップ、`package-lock.json` の更新後、クリーンな再構築。
> - **`npm install`**: 新しいパッケージの追加や既存パッケージのアップグレード時のみ使用。

このセクションでは、開発用および本番用の拡張機能のビルド方法について説明します。

### 開発（ウォッチモード）
効率的な開発のために、ウォッチモードを使用できます。ソースファイルを変更するたびに、拡張機能が自動的に再ビルドされます。

> [!NOTE]
> ウォッチモードで生成されたファイルは開発目的のみです。本番用に最小化や最適化はされていません。公開リリースには必ず本番用ビルドを使用してください。

- **Chrome用**:
  ```bash
  npm run dev:chrome
  ```
- **Firefox用**:
  ```bash
  npm run dev:firefox
  ```
> [!NOTE]
> 開発モードは Firefox MV3 をサポートしていません。代替案については、https://github.com/wxt-dev/wxt/issues/230#issuecomment-1806881653 を参照してください。

### 本番用 / 手動ビルド
本番用ビルドを作成する場合や、テストのために手動で 1 回ビルドを行う場合は、以下のコマンドを使用します。

- **Chrome用**
    - **開発環境向け**: `npm run build:chrome:dev` (ソースマップあり)
    - **本番環境向け**: `npm run build:chrome:prod` (ソースマップなし)
- **Firefox用**
    - **開発環境向け**: `npm run build:firefox:dev` (ソースマップあり)
    - **本番環境向け**: `npm run build:firefox:prod` (ソースマップなし)

> [!NOTE]
> ソースマップの有無は `wxt.config.ts` の `build.sourcemap` 設定で制御されます。これは `wxt` コマンドに渡される `--mode` フラグ（例: `--mode sourcemap`）に基づいて切り替わります。

## インストール（パッケージ化されていない拡張機能の読み込み）

ビルドによって生成された拡張機能をインストールするには、以下の手順に従ってください。ビルドされた拡張機能は自動的には更新されないことに注意してください。

### Chrome および Chromium ベースのブラウザの場合

1. 新しいタブで `chrome://extensions` を入力するか、ツールバーから「拡張機能を管理」ページを開きます。
2. 「デベロッパー モード」の横にあるトグルスイッチをクリックして有効にします。
3. 「パッケージ化されていない拡張機能を読み込む」ボタンをクリックし、ビルドした拡張機能のディレクトリを選択します。
4. 設定が有効であれば、拡張機能がインストールされます。

### Firefox および Firefox ベースのブラウザの場合

1. 新しいタブで `about:debugging` を入力します。
2. 「この Firefox」をクリックします。
3. 「一時的なアドオンを読み込む...」ボタンをクリックします。
4. ビルドディレクトリ内の `manifest.json` ファイル（例: `.output/firefox-mv3/manifest.json`）を選択します。
5. マニフェストが有効であれば、アドオンが一時的に読み込まれます。
