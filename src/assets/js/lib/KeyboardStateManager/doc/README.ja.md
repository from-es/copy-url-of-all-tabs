# KeyboardStateManager.ts

キーボード入力を管理するための汎用的な抽象化レイヤーです。`hotkeys-js` をラップすることで依存関係を隔離し、キーの押下状態を追跡するためのクリーンなステートベースのインターフェースを提供します。

## 概要

キーボードショートカットや修飾キー（Alt、Shift、Ctrlなど）の状態管理は、特にリアクティブなUIを維持しようとする場合に複雑になりがちです。`KeyboardStateManager.ts` は、「追跡対象」として明示的に指定されたキーの現在押されている状態を `Set` オブジェクトで保持することで、このプロセスを簡素化します。

このクラスはフレームワークに依存しませんが、明示的な同期ポイントを提供することで、リアクティビティシステム（SvelteのrunesやVueのrefsなど）に簡単に統合できるように設計されています。

## 主な機能

- **依存関係の隔離:** `hotkeys-js` をラップしているため、アプリケーションの他の部分が直接 `hotkeys-js` に依存する必要がありません。
- **明示的なスコープ:** `setup()` を通じて**明示的に指定したキーのみ**を追跡します。
- **手動同期:** `sync()` メソッドにより、状態が更新されるタイミングを正確に制御できます。これは、不要なリアクティビティのオーバーヘッドや、イベントバブリング中のレースコンディションを避けるのに有効です。
- **安全なクリーンアップ:** `teardown()` を提供し、すべてのグローバルイベントリスナーを解除します。これにより、マルチページやコンポーネントベースの環境でのメモリリークやハンドラの衝突を防ぎます。
- **フレームワーク非依存:** `hotkeys-js` 以外の外部依存関係を持たない、純粋な TypeScript クラスです。
- **責務の限定:** このライブラリの役割は「キー入力状態の管理（どのキーが押されているか）」に特化しています。特定のキー入力に応じて何らかの処理（コールバックの実行など）を行う機能は提供していません。アクションの実行は、利用側で `status` の変化を監視して実装してください。

## 基本的な使い方

以下に、マネージャーの基本的な使用例を示します。

```typescript
import { KeyboardStateManager } from './KeyboardStateManager';

const kbd = new KeyboardStateManager();

// 1. 追跡したいキーを定義する
kbd.setup(['w', 'h', 's']);

// 2. イベントに応じて状態を同期する（例：グローバルな window リスナー内）
window.addEventListener('keydown', () => kbd.sync());
window.addEventListener('keyup', () => kbd.sync());

// 3. 状態を確認する
if (kbd.status.has('w')) {
    console.log('Wキーが現在押されています');
}

// 4. 終了時にクリーンアップする
kbd.teardown();
```

## APIリファレンス

### `status: Set<string>`

現在押されている追跡対象キーの識別子を含む `Set` オブジェクトです。
注意: このセットはインプレースで変更されます。リアクティビティシステムを使用している場合は、このプロパティをラップするか、`sync()` を呼び出した後に更新をトリガーする必要があります。

### `setup(keys: string[])`

追跡するキーの範囲を設定します。

- **`keys: string[]`**: キー識別子の配列（例：`"alt"`, `"shift"`, `"a"`, `"control"`）。
- **動作**: 
    - 以前に登録されたハンドラをすべて解除します。
    - キーを小文字に正規化し、重複を削除します。
    - 指定されたキーに対して `hotkeys-js` を使用して内部ハンドラを登録します。

### `sync()`

現在の物理的なキーの状態を `status` セットに同期します。

- **動作**: 
    - すべての追跡対象キーをループし、`hotkeys-js` を介して現在の状態を確認します。
    - 押されているキーを `status` セットに追加し、離されたキーを削除します。
    - キーが押されたとき、または離されたときに、デバッグ情報をコンソールに出力します。

### `reset()`

`status` セットを強制的にクリアします。

- **ユースケース**: ウィンドウがフォーカスを失った場合や、予期しない状態変化が発生したときに状態をリセットするのに便利です。

### `teardown()`

追跡を完全に停止し、リソースをクリーンアップします。

- **動作**:
    - `hotkeys-js` からすべてのハンドラを解除します。
    - `status` セットをクリアします。
    - 追跡対象キーの内部リストをリセットします。

## 実装の詳細

### リアクティビティへの統合

`status` は標準の `Set` オブジェクトであるため、デフォルトではリアクティブではありません。Svelte 5 などのフレームワークでは、次のようにラップすることを検討してください。

```typescript
const kbd = new KeyboardStateManager();
let keyStatus = $state(kbd.status);

function handleKey() {
    kbd.sync();
    keyStatus = new Set(kbd.status); // リアクティビティをトリガー
}
```
