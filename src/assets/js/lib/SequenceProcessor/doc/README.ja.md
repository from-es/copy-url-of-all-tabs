# SequenceProcessor ライブラリ

**最終更新日:** 2026年6月20日

## 概要

`SequenceProcessor` は、データオブジェクトに対して一連の処理（ルール）を順番に適用するための汎用ライブラリです。

データの変換、状態の更新、あるいは段階的なバリデーションなど、複数の独立したロジックを特定の順序で実行し、その過程でのエラーハンドリングや実行条件の制御を統合的に管理します。

型安全なジェネリクス (`T`, `C`) を採用しており、あらゆるデータ構造と実行コンテキストに対応可能な柔軟な設計となっています。

## 主な機能

1. **シーケンシャルな実行:**
	- 登録されたルールを `order`（実行順序）に従って順番に適用します。
2. **高度なフィルタリング (Spec):**
	- 実行プラットフォーム、バージョン（ライフサイクル）、有効/無効フラグに基づき、実行時にルールを自動的にフィルタリングします。
3. **柔軟なエラー制御:**
	- ルールごとに「致命的 (Critical)」かどうかを設定でき、エラー発生時に処理を継続するか、あるいは安全のために全ロールバックするかを選択できます。
4. **コンテキストの注入:**
	- すべてのルールに対し、共通の「コンテキスト (C)」を渡すことができ、環境情報や外部状態に依存した判定が可能です。
5. **不変性の保護 (Immutability):**
	- 入力データを保護するため、内部でクローンを作成して処理を行います。エラー発生時のロールバック機能も備えています。

## ディレクトリ構成とインポート関係

本ライブラリおよびルール群は、以下のディレクトリ構造を前提としています。ドキュメント内のコードブロックは、この配置における相対パス指定でインポートを記述しています。

```text
src/
└── assets/
    └── js/
        ├── app/
        │   └── SequenceProcessor/
        │       ├── context.ts
        │       ├── types.ts
        │       └── rules/
        │           ├── index.ts              (インポート元: ../../lib/SequenceProcessor/loadRules)
        │           └── migrate/
        │               └── v1.rules.ts       (インポート元: ../../../../lib/SequenceProcessor/types)
        └── lib/
            ├── CloneObject.ts
            └── SequenceProcessor/
                ├── index.ts                  (ライブラリ本体)
                ├── loadRules.ts
                ├── types.ts
                └── MigrationManager.ts
```
